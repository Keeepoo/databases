# Базы данных: решение задач на SQL

## Описание проекта
Данный репозиторий содержит решения 13 задач по предмету «Базы данных».  
Задания охватывают четыре базы данных:
1. Транспортные средства
2. Автомобильные гонки
3. Бронирование отелей
4. Структура организации

## Структура репозитория
- `vehicle_db/` – база «Транспортные средства» (2 задачи)
- `racing_db/` – база «Автомобильные гонки» (5 задач)
- `hotel_db/` – база «Бронирование отелей» (3 задачи)
- `organization_db/` – база «Структура организации» (3 задачи)

Каждая папка содержит:
- `schema.sql` – скрипт создания таблиц (MySQL)
- `data.sql` – скрипт наполнения тестовыми данными
- `task1.sql`, `task2.sql`, ... – решения задач

## Требования к окружению
- MySQL 8.0+ (или MariaDB 10.2+)
- Любой клиент: phpMyAdmin, MySQL Workbench, DBeaver и т.д.

## Цели и задачи
- Научиться проектировать реляционные схемы и задавать связи через внешние ключи.
- Освоить операции соединения таблиц (`INNER JOIN`, `LEFT JOIN`).
- Использовать агрегатные функции (`AVG`, `COUNT`, `SUM`, `GROUP_CONCAT`) и группировку.
- Применять `UNION` и `UNION ALL` для объединения данных из разных таблиц.
- Писать рекурсивные запросы (`WITH RECURSIVE`) для иерархических структур.
- Вычислять сложные метрики (средняя позиция, количество подчинённых, предпочтения клиентов и т.д.).
- Формировать итоговые отчёты с сортировкой и фильтрацией.

## Структура баз данных и основные функции

### 1. Транспортные средства (vehicle_db)
Таблицы:
- `Vehicle` – производитель, модель, тип (Car/Motorcycle/Bicycle)
- `Car` – VIN, объём двигателя, мощность, цена, трансмиссия
- `Motorcycle` – VIN, объём, мощность, цена, тип мотоцикла
- `Bicycle` – серийный номер, количество передач, цена, тип велосипеда

**Реализованные функции:**
- Задача 1: отбор мотоциклов по мощности, цене и типу с сортировкой.
- Задача 2: объединение автомобилей, мотоциклов и велосипедов с разными наборами атрибутов (NULL для велосипедов).

### 2. Автомобильные гонки (racing_db)
Таблицы:
- `Classes` – класс автомобиля, тип, страна, число дверей, объём двигателя, вес
- `Cars` – название автомобиля, класс, год выпуска
- `Races` – название гонки, дата
- `Results` – связь автомобиля и гонки с позицией финиша

**Реализованные функции:**
- Задача 1: для каждого класса автомобиль с наименьшей средней позицией.
- Задача 2: абсолютный лидер по средней позиции (с учётом алфавита при равенстве).
- Задача 3: классы с наилучшей средней позицией – вывод всех их автомобилей и общего количества гонок класса.
- Задача 4: автомобили, чья средняя позиция лучше средней по классу (в классе от 2 авто).
- Задача 5: классы с наибольшим числом автомобилей, имеющих среднюю позицию >3.0.

### 3. Бронирование отелей (hotel_db)
Таблицы:
- `Hotel` – отель (ID, название, расположение)
- `Room` – номер (ID, ID отеля, тип, цена, вместимость)
- `Customer` – клиент (ID, имя, email, телефон)
- `Booking` – бронирование (ID, ID номера, ID клиента, даты заезда/отъезда)

**Реализованные функции:**
- Задача 1: клиенты с более чем двумя бронированиями в разных отелях – список отелей, средняя длительность пребывания.
- Задача 2: пересечение клиентов, соответствующих условиям (>2 броней в >1 отеле) и потративших более 500 USD.
- Задача 3: категоризация отелей (дешёвый/средний/дорогой) и определение предпочитаемого типа для каждого клиента.

### 4. Структура организации (organization_db)
Таблицы:
- `Departments` – отделы
- `Roles` – роли (менеджер, директор и др.)
- `Employees` – сотрудники (самореференция для иерархии)
- `Projects` – проекты (привязаны к отделу)
- `Tasks` – задачи (назначены сотруднику, относятся к проекту)

**Реализованные функции (с использованием рекурсивных CTE):**
- Задача 1: все подчинённые Ивана Иванова (включая косвенных) с проектами и задачами.
- Задача 2: то же + количество задач и количество прямых подчинённых.
- Задача 3: все менеджеры, имеющие хотя бы одного подчинённого (включая косвенных), с проектами и задачами.


## Инструкция по запуску (на примере vehicle_db)
1. Создайте базу данных, например `vehicle_db`.
2. Выполните в ней `schema.sql` для создания таблиц.
3. Выполните `data.sql` для заполнения данными.
4. Выполните любой `task.sql` для получения результата.

Аналогично для других баз.
